# Workout App V2 — Design Spec
_Date: 2026-05-11_

## What we're building

A set of targeted upgrades to the live workout app (github.com/ABKodes/workout-app) that make the gym experience seamless — no keyboard, no thinking about what's next — and motivating via visible progress over time. All data stays in localStorage (no backend).

---

## Problems being solved

1. **Logging friction** — current text inputs require a keyboard mid-set with sweaty hands
2. **No flow guidance** — user has to manually find the next exercise/set
3. **No motivation loop** — no way to see weights going up or body weight going down over weeks

---

## V2 Feature Set

| # | Feature | Solves |
|---|---|---|
| 1 | Stepper logging (+/− buttons, "same as last time") | Friction |
| 2 | Guided session mode (set-by-set full screen) | Friction + Flow |
| 3 | Body weight tracker + trend chart | Motivation |
| 4 | Strength progress chart per exercise | Motivation |
| 5 | PR detection + celebration overlay | Motivation |
| 6 | Auto-progression tip after session | Flow |
| 7 | Warm-up sets calculator | Flow |
| 8 | Log export / import (backup) | Data safety |

## Architecture changes

### Tab bar
**V1:** Mon · Tue · Wed · Thu · Fri · Sat · Sun · NUT (8 tabs)
**V2:** Mon · Tue · Wed · Thu · Fri · Sat · Sun · STATS (8 tabs)

The NUT tab becomes a STATS tab (📊). Inside Stats, a pill sub-nav switches between three sections: **Nutrition**, **Body Weight**, and **Progress**. Keeps tab count at 8 and scales to any future stats screens.

## Feature specs

### 1. Stepper Logging (SetRow rewrite)

**Replaces:** two text `<input>` fields for weight and reps.

**New UI per set:**
- "Same as last time" pill button — visible only when prevLog exists. One tap fills weight + reps from last session and marks set done + starts timer.
- Weight stepper: `−` | `55 kg` | `+` — step size 2.5 kg. Pre-loaded from prevLog.
- Reps stepper: `−` | `12` | `+` — step size 1. Pre-loaded from prevLog.
- Done ✓ button — unchanged.

**Behaviour:**
- Values start at prevLog values. If no prevLog exists, weight defaults to 20 (bar weight) and reps defaults to the lower bound of the exercise's reps range.
- User only has to tap +/− to adjust, or tap "same" to confirm unchanged.
- No keyboard ever appears.
- Weight decrements minimum 0, reps minimum 1.

---

### 2. Guided Session Mode (GuidedSession.tsx)

**Entry:** "🎯 Start Guided Session" button at top of DayScreen for gym days only. Renders as a full-screen fixed overlay (z-60).

**State:** `currentExIdx` (0…n exercises) + `currentSetIdx` (0…n sets).

**Screen layout (top to bottom):**
1. Top bar: progress bar (% of total sets done), `✕` exit button top-right
2. Exercise context: "Exercise 3 of 6" label, exercise name (large), exercise note (small grey)
3. Set pills: all sets for this exercise shown as small pills (done = green, current = orange, upcoming = grey)
4. Big steppers: weight (step 2.5kg) and reps (step 1), pre-loaded from prevLog
5. "Same as last time" pill if prevLog exists
6. Large primary button: "✓ Done — start rest timer"
7. "What's next" preview line: "After rest: Set 2 of Lat Pulldown" or "After rest: DB RDL"

**Flow:**
- Tap done → logs set → starts RestTimer
- RestTimer finishes → auto-advances to next set (or next exercise if last set)
- If all exercises done → shows "Session complete 🔥" screen with summary (total sets, PRs hit, session note input, Finish button)
- Tapping ✕ exits guided mode back to normal DayScreen (log data is already saved)

**Does not replace** normal DayScreen — user can ignore guided mode and use the regular screen. Both read/write to the same `useLog` hook.

---

### 3. Body Weight Tracker (BodyWeightSection.tsx + useBodyWeight.ts)

**Data model:**
```typescript
interface BodyWeightEntry {
  date: string   // "2026-05-11"
  weight: number // kg
}
// localStorage key: "bwlog_v1" → BodyWeightEntry[]
```

**Screen layout:**
1. Three stat cards: Today's weight · Total lost (kg) · Rate (kg/week, rolling 2-week average)
2. SVG line chart: last 5 weeks. Two lines — actual weight (green) + target trend (orange dashed, −0.6kg/week from start weight)
3. "Log today's weight" button → inline input + confirm (no modal, expands in place). Only one entry per day (overwrites if re-logged same day).
4. Recent entries list: date · weight · delta from previous (green = loss, red = gain)

**Rate calculation:** `(earliestWeight - latestWeight) / weeks elapsed`. Shown as negative number (loss) in purple.

---

### 4. Strength Progress Charts (ProgressSection.tsx + useProgress.ts)

**Screen layout:**
1. Scrollable exercise picker pills — only shows gym-day exercises (excludes football/cardio days)
2. PR badge: "🏆 Best: 62.5kg × 10" + date
3. SVG line chart: x = session date, y = heaviest set weight logged that session. Orange line + gradient fill. Star marker on PR session.
4. Session history bars: horizontal bar per session, coloured by recency (darker = older). Bar length = weight relative to PR. Shows date + weight label.

**Data derivation (useProgress.ts):**
- Reads all SessionLog entries for a given exercise name
- Extracts heaviest `weight` value across all done sets per session
- Sorts by date ascending
- PR = maximum weight ever logged for that exercise

**Exercise picker default:** first gym-day exercise in data.ts (Lat Pulldown for Tue, DB Lateral Raise for Mon, etc.) based on currently active day.

---

### 5. PR Detection + Celebration (PrCelebration.tsx)

**Trigger:** in `DayScreen` and `GuidedSession`, after calling `logSet`, compare the newly entered weight against `useProgress(exerciseName).prWeight`. If the new weight exceeds the previous PR, set `newPR` state. PR detection happens at the component level (not inside the hook) to keep `useLog` focused on CRUD only.

**Overlay:** full-screen dark overlay, centre: large 🏆 emoji + "New PR!" text + weight + CSS burst animation (scale + opacity keyframes, no library). Auto-dismisses after 2.5 seconds. Can also tap to dismiss.

---

### 6. Auto-progression Tip

**Trigger:** shown in the "Session complete" screen (both guided mode and normal Finish session button).

**Logic:** for each exercise in the session, if every logged set hit or exceeded the target reps, suggest "+2.5kg next time." Parse the upper bound from `exercise.reps` using: take the last integer in the string (e.g. "10–12" → 12, "8–10" → 10, "8" → 8). If the user's logged reps ≥ upper bound on every set, the progression tip fires.

**UI:** simple list below the session complete message: "💡 Lat Pulldown — you hit all reps. Try 62.5kg next session."

---

### 7. Warm-up Sets Calculator (WarmupCalc.tsx)

**Entry:** small "🔥 warm-up" button added to ExerciseCard button row (alongside existing 1RM and plates buttons). Opens as bottom-sheet modal (same pattern as RestTimer).

**Input:** working weight (pre-filled from last session's logged weight, or 0).

**Output table:**
| Set | % | Weight | Reps |
|---|---|---|---|
| 1 | 50% | 27.5 kg | 8 |
| 2 | 70% | 37.5 kg | 5 |
| 3 | 85% | 47.5 kg | 3 |

Weights rounded to nearest 2.5kg. Only shown for gym-day exercises (restSeconds > 0).

---

### 8. Log Export / Import (ExportImport.tsx)

**Location:** bottom of StatsScreen (below all sections), always visible.

**Export:** "⬇ Download backup" → JSON.stringify of all localStorage keys (`wlog_v1`, `bwlog_v1`) → triggers `<a download>` with filename `workout-backup-YYYY-MM-DD.json`.

**Import:** "⬆ Restore backup" → `<input type="file" accept=".json">` → reads file → merges entries by date (import wins on conflict) → saves back to localStorage → shows "✓ Restored N sessions."

---

## StatsScreen container (StatsScreen.tsx)

Pill sub-nav at top: `Nutrition | Body Weight | Progress`

- Nutrition section: unchanged from v1 NutritionScreen content (macros, tip, supplements, 1RM calc, plate calc)
- Body Weight section: BodyWeightSection component
- Progress section: ProgressSection component + ExportImport at bottom

Default active pill: Progress (most motivating screen to land on).

---

## Data safety

- All data stays in localStorage. No network requests.
- Export button is the only backup mechanism — user should download periodically.
- Import merges by date key, never destructively overwrites unless same-day conflict.

---

## Out of scope for V2
- Cloud sync
- Push notifications / reminders
- Sharing PRs to social media
- Multiple users
- Exercise video library
- Changing the workout program itself (edit exercises/sets/reps)
